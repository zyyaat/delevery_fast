STACK

Technology Stack / التقنيات المستخدمة

Performance-first Go backend, React across all clients, PostgreSQL as the source of truth.

memory
1 · BACKEND
PERFORMANCE-CRITICAL
→Go 1.22 — chi/echo router, goroutines for concurrency
→gRPC for inter-service communication
→Protocol Buffers for typed contracts
Go chosen for low-latency, high-concurrency — handles 50K orders/day on modest infra.
devices
2 · FRONTEND
CROSS-PLATFORM
→React Native — Customer, Driver, Restaurant, Supervisor apps
→React + Vite — Support, Command Center, Employee Portal (web only)
→Tailwind CSS + shadcn/ui for design system
React knowledge reused across mobile + web — single team can ship to all 7 clients.
storage
3 · DATABASE
TRANSACTIONAL + GEOSPATIAL
→PostgreSQL 16 — primary OLTP, partitioned by city
→PostGIS extension — geospatial queries for driver matching
→Redis 7 — cache, session, geospatial index via GEOADD
PostgreSQL + PostGIS handles 95% of needs; Redis for sub-ms geospatial lookups.
stream
4 · EVENT BUS
ASYNC BACKBONE
→Apache Kafka — order events, driver location streams
→Schema Registry — Avro for typed events
→Kafka Streams / Flink — real-time aggregations
Kafka decouples services and provides replay for fraud analytics + order history.
cloud
5 · INFRASTRUCTURE
CONTAINERIZED
→Kubernetes — EKS self-managed on AWS
→Docker — multi-stage builds, distroless images
→Terraform — IaC, multi-env
→ArgoCD — GitOps deploys
K8s gives horizontal autoscaling per service — critical for peak-hour spikes.
visibility
6 · OBSERVABILITY
THREE PILLARS
→Prometheus + Grafana — metrics
→Loki — logs
→Jaeger / OpenTelemetry — traces
→Sentry — client-side errors
Single observability stack across all 12 services and 7 clients.
