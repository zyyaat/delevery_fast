ADR SUMMARY

Technology Decisions — Rationale / مبررات القرارات التقنية

Why each major tech choice was made — alternatives considered, decision, revisit triggers.

1Backend Language: Go
ALTNode.js, Python, Java, Rust
CHOICEGo 1.22
WHYBest concurrency model for I/O-heavy services; fast compile times enable rapid iteration; small binary size = fast container starts; strong standard library for HTTP/gRPC.
REVISITIf we add ML inference to backend → Python sidecar for ML; Go stays for business logic.
2Frontend Framework: React (Native + Web)
ALTVue, Svelte, Flutter, Swift+Kotlin native
CHOICEReact 18 + React Native 0.74
WHYSingle component mental model across mobile + web; largest talent pool in Egypt; ecosystem maturity; React knowledge reusable.
REVISITIf performance becomes an issue on low-end Android → consider Flutter for Customer App only.
3Database: PostgreSQL
ALTMySQL, MongoDB, DynamoDB
CHOICEPostgreSQL 16 + PostGIS
WHYACID guarantees for orders/payments; JSONB for flexible metadata; PostGIS for geospatial; mature ecosystem; battle-tested at scale (Uber, Instagram).
REVISITNever for transactional data. May add ClickHouse for analytics (planned).
4Event Bus: Apache Kafka
ALTRabbitMQ, NATS, Redis Streams, AWS Kinesis
CHOICEApache Kafka 3.x (MSK or self-managed)
WHYIndustry standard for event-driven; replay capability for fraud analytics; exactly-once semantics for payments; massive scale proven (LinkedIn, Uber, Netflix).
REVISITIf operational burden too high → consider Upstash Kafka (managed) or Redpanda (Kafka-compatible, lower ops).
5Container Orchestration: Kubernetes
ALTECS, Nomad, Docker Swarm, serverless (Lambda)
CHOICEKubernetes (EKS)
WHYPortability (no vendor lock-in); rich ecosystem (Helm, ArgoCD, KEDA); standard skill for engineers; horizontal pod autoscaling critical for peak hours.
REVISITIf team too small to operate K8s → ECS Fargate for first 6 months, migrate to EKS in Phase 2.
6Mobile Architecture: React Native
ALTNative (Swift+Kotlin), Flutter, PWA
CHOICEReact Native 0.74 (New Architecture enabled)
WHYSingle codebase for iOS+Android; same engineers can ship to both; fast iteration via hot reload; reusable components with web app; Hermes engine performance now comparable to native for our use case.
REVISITIf driver app performance issues (GPS, background tasks) → consider going native for Driver App only.
