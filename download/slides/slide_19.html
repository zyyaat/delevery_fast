Authentication & Authorization
Validates JWT on every request, injects user_id and roles into upstream headers. Supports API keys for restaurant-to-platform integrations.
2
Rate Limiting
Per-IP and per-user limits — 100 RPS default, 1000 RPS for restaurant API keys. Burst capacity for peak hours. Backed by Redis token bucket.
3
Routing & Versioning
Path-based routing to 12 backend services. Supports /v1/, /v2/ versioning for backwards compatibility. Canary deployments via weighted routing.
4
TLS Termination & WAF
TLS 1.3 termination at the edge. Cloudflare WAF rules for OWASP top 10. Bot detection for fake order attempts.
5
Request Aggregation
BFF (Backend-For-Frontend) pattern — Customer App's /home endpoint aggregates restaurant list + promotions + user profile in one call. Reduces client round trips from 5 to 1.
