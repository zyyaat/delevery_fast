Build Scope

/ نطاق البناء
Apps
  • Customer App (React Native) — restaurant browse, cart, checkout, order tracking
  • Restaurant App (React Native) — incoming order alerts, accept/reject, status update
  • Driver App (React Native) — order alerts, accept, navigation, mark delivered
Services
  • Auth Service (phone+OTP, JWT)
  • Restaurant Catalog (50 hand-onboarded)
  • Menu Service (static menus, no modifiers v1)
  • Order Service (state machine, 6 statuses)
  • Payment Service (Vodafone Cash only)
  • Notification Service (push + SMS)
Infra
  • Single PostgreSQL instance (1 primary + 1 replica)
  • Redis (cache + sessions)
  • Kafka (3 brokers, basic topics)
  • EKS cluster (3 nodes)
  • Cloudflare CDN

Exit Criteria

/ معايير الانتقال
Metrics
  • 500 orders/day average (last 7 days)
  • 50 restaurants live and active
  • p95 API latency <3s
  • 99% uptime (excluding planned maintenance)
  • Payment success rate >95%
  • Order completion rate >90% (not cancelled)
Soft gates
  • Restaurant NPS >30
  • Customer repeat rate >25% (orders from returning users)
  • Driver retention >70% (week-over-week)
  • No P0 bugs open for >48h
timer If exit criteria not met → 2-week extension before Phase 2 kickoff.
