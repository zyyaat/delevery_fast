Database Architecture / معمارية قاعدة البيانات

12 service-owned databases — polyglot persistence tuned per domain.

Database-per-Service / قاعدة لكل خدمة

Each of the 12 microservices owns its own PostgreSQL logical database. No service reads another's database directly — only via API or event. This enforces domain boundaries and lets each service optimize its schema independently. Cross-service joins are replaced by either API composition (for reads) or event-carried state transfer (for writes).

Partitioning Strategy / استراتيجية التقسيم

Orders table partitioned by month (range partitioning) — keeps active partitions small. Driver_locations archived to S3 after 7 days. Restaurant data sharded by city for >10K restaurants — Cairo/Giza/Alexandria/Mansoura each get a shard.
