PLATFORM ENGINEERING

DevOps & Infrastructure / العمليات والبنية التحتية

Kubernetes-native, GitOps-deployed, fully observable — built for 99.9% uptime from day one.

hub
Container Orchestration
  • Kubernetes (EKS on AWS)
  • 3 node groups: backend (Go), web (React), data (Postgres/Redis/Kafka)
  • Autoscaling: HPA on CPU/RPS, KEDA for Kafka-driven scaling
  • Pod anti-affinity for HA across AZs
  • Cluster autoscaler for node-level scaling
TARGET
Spin up new pod in <30s; auto-recover from node failure in <2min.
sync
CI/CD Pipeline
  • GitHub Actions for CI (test, lint, build, scan)
  • ArgoCD for CD (GitOps — cluster state declared in Git)
  • Helm charts per service (versioned, reusable)
  • Image scanning via Trivy (block on CRITICAL CVEs)
  • Deploy preview env per PR (ephemeral, auto-destroy after 24h)
PIPELINE
push → lint → test → build → scan → staging → smoke → prod (manual approval).
code
Infrastructure as Code
  • Terraform for AWS resources (VPC, EKS, RDS, S3, etc.)
  • Terragrunt for multi-env (dev/staging/prod) DRY
  • Terraform state in S3 + DynamoDB lock
  • All changes via PR review — no console clicks
  • Drift detection nightly
POLICY
If it's not in Terraform, it doesn't exist.
visibility
Observability
  • Prometheus + Grafana (metrics, 15s scrape, 90-day retention)
  • Loki (logs, structured JSON, 30-day retention)
  • Jaeger + OpenTelemetry (traces, 7-day retention, 10% sampling)
  • Sentry (client + backend errors)
  • Alerting: PagerDuty for P0/P1, Slack for P2/P3
SLOs
99.9% uptime for Order/Payment; 99.5% for others.
vpn_key
Secrets & Config
  • AWS Secrets Manager for production secrets
  • External Secrets Operator syncs to K8s secrets
  • Sealed Secrets for GitOps-friendly sensitive values
  • Rotation: DB creds every 90 days, API keys quarterly
  • No secrets in code, env files, or chat
AUDIT
Every secret access logged; quarterly access review.
backup
Disaster Recovery
  • RPO: 5 min (Postgres WAL streaming + 5-min snapshots)
  • RTO: 30 min (multi-AZ failover, tested quarterly)
  • Cross-region replication (us-east-1 → us-west-2)
  • Quarterly DR drill (full restore in staging)
  • Runbooks for top 20 incident scenarios
TEST
Chaos game days quarterly — kill pods, simulate AZ failure.
