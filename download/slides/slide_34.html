Internal Employee Portal / بوابة الموظفين الداخلية

Onboarding, biometric auth, and audit trail — the controls that prevent the hardest fraud to detect.

apps

Portal Modules

/ وحدات البوابة
1
Employee Onboarding
HR creates account → employee sets password + TOTP → biometric enrollment (fingerprint/face via WebAuthn). Background check status tracked. Access provisioning automated based on role.
2
Sensitive Action Authentication
When employee triggers a sensitive action (refund >EGP 200, restaurant approval, payout), portal prompts for biometric re-auth. WebAuthn standard — fingerprint/face/camera. No password reuse.
3
Audit Log (Immutable)
Every action written to audit_logs table — append-only. Hash-chained: each record includes hash of previous record. Tampering detected via chain validation. Logs exported daily to cold storage (S3 Object Lock).
4
Dual Approval Workflow
High-value actions require 2 approvers from different teams. Refund >EGP 500 → support_l2 + ops_manager. Restaurant onboarding → ops_manager + finance. Driver KYC approval → field_supervisor + ops_manager.
5
Quarterly Access Review
Command Center runs access review every quarter. Managers must confirm or revoke each team member's permissions. Stale accounts (>30 days inactive) auto-suspended.

TECHNICAL BLUEPRINT v1.0
