Auth Stack

/ نظام المصادقة
person

Customer / Driver Auth

  • Phone + OTP (SMS via Twilio/local provider)
  • Optional password / biometric on device
  • JWT access token (15 min TTL) + refresh token (30 days)
  • Refresh rotation — old token invalidated on use
  • Device binding — refresh token tied to device fingerprint
restaurant

Restaurant Auth

  • Phone + OTP for primary account
  • Sub-accounts (manager, cashier) with role scoping
  • API keys for POS integration (rotated quarterly)
  • IP allowlist for API keys (configurable per restaurant)
admin_panel_settings

Internal Employee Auth

  • Username + password + TOTP (mandatory 2FA)
  • Biometric (fingerprint/face) for: refunds >EGP 200, restaurant approval, driver payout, menu overrides
  • Hardware security key (YubiKey) for Command Center admins
  • Session timeout: 15 min idle, 8h max

TECHNICAL BLUEPRINT v1.0
- Three auth stacks: Customer/Driver (OTP+JWT), Restaurant (sub-accounts+API keys), Internal (TOTP+biometric+YubiKey). - Refresh token rotation + device binding prevent replay attacks. - RBAC matrix: 6 roles × 5 permissions — partials are amount-scoped. - super_admin is the only full-access role. - Central Authorization Service — no app talks to DB directly; everything logged.
